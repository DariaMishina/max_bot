# Скрипт Метрики на лендинге (Директ → лендинг → бот)

Счётчик на странице: **107059457** (подключён в настройках сайта).

## Что делает скрипт

- Отправляет цели в Метрику при клике: `go_to_tg`, `go_to_max`, `go_to_messenger` (последняя — для оптимизации в Директе).
- Подставляет в ссылки на TG и на MAX параметр `start` в формате `__client_id__{ClientID}__camp_{utm_campaign}`, чтобы в боте сохранялись атрибуция и UTM.

## Использование

1. В интерфейсе лендинга (Тильда и т.п.) должен быть подключён счётчик Яндекс.Метрики **107059457**.
2. Скрипт из файла `landing_metrika_script.js` вставьте в код страницы (например, блок «Действия» / кастомный JS / перед `</body>`).

Полный готовый код — в `landing_metrika_script.js`. Если у вас уже был старый вариант скрипта, замените его на содержимое этого файла.

## Отличия от предыдущей версии

- Добавлена функция **`updateMaxLinks(clientId)`**: все ссылки, ведущие на `max.ru/`, получают `?start=__client_id__...__camp_...` (как и ссылки на TG).
- В **init** после получения ClientID вызываются и `updateTelegramLinks(clientId)`, и **`updateMaxLinks(clientId)`**.
- Ссылки на MAX ведут в бота с параметром, который max_bot разбирает и сохраняет в БД (`client_id`, `utm_campaign`, `source=landing`).

## Цели в Метрике (107059457)

| Идентификатор цели  | Назначение                          |
|---------------------|-------------------------------------|
| `go_to_tg`          | Клик по ссылке в Telegram          |
| `go_to_max`         | Клик по ссылке в MAX                |
| `go_to_messenger`   | Любой переход в мессенджер (для Директа) |

В кампании Директ для оптимизации укажите цель **`go_to_messenger`**.
