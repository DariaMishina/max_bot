# Рассылки бота Сфера Таро

Документ описывает автоматические и ручные рассылки, логику сегментации и расписание.

---

## Обзор автоматических рассылок

| Рассылка | Расписание | Файл | Функция |
|----------|------------|------|---------|
| Карта дня | Ежедневно 09:25 MSK | `handlers/daily_card.py` | `send_daily_card_to_all_users()` |
| Welcome-активация | Ежедневно 10:00–20:00 MSK, тик 30 мин | `send_message.py` | `send_activation_broadcast()` |
| Пн/Чт сегментированная | Пн и Чт 10:00–20:00 MSK, тик 30 мин | `send_message.py` | `send_divination_reminder_broadcast()` |
| Напоминания об оплате | Каждые 2 мин (10м / 1ч / 3ч после платежа) | `main/payment_reminders.py` | `process_payment_reminders()` |

Ручные рассылки — через CLI `send_message.py --help`.

---

## Расписание отправки (окно 10:00–20:00 MSK)

Рассылки **активации** и **Пн/Чт** не отправляются всем в одну минуту. Логика в `main/broadcast_schedule.py`:

1. **Окно:** 10:00–20:00 MSK (600 минут).
2. **Тик планировщика:** каждые 30 минут (`:00` и `:30`).
3. **Персональный слот пользователя:**
   - если есть `last_active_at` — минута отправки ≈ час последней активности (если вне окна — маппинг внутрь окна);
   - иначе — `user_id % 600` минут от 10:00 (стабильный слот от недели к неделе).
4. **Отправка:** в тик, когда `send_minute <= now < send_minute + 30`.

## 1. Welcome-активация

**Цель:** вернуть пользователей, которые зашли в бота, но ни разу не сделали расклад.

| Параметр | Значение |
|----------|----------|
| Триггер | ≥ 24 часа после `created_at` |
| Условия | `activation_sent_at IS NULL`, нет записей в `max_divinations`, не заблокирован, нет открытой оплаты |
| Отправка | Один раз, в персональный слот 10:00–20:00 MSK |
| После отправки | `activation_sent_at = NOW()` |

**Без paywall** — только приглашение сделать первый расклад.

**Anti-spam при деплое:** пользователям старше 7 дней без гаданий проставляется `activation_sent_at`, чтобы не слать активацию «задним числом» (см. `migrations/20260531_broadcasts.sql`).

---

## 2. Пн/Чт сегментированная рассылка

**Цель:** персонализированное напоминание в зависимости от статуса пользователя.

| Параметр | Значение |
|----------|----------|
| Дни | Понедельник и четверг |
| Окно | 10:00–20:00 MSK, тик 30 мин |
| Повтор в тот же день | Нет (`last_div_reminder_broadcast_at`) |

### Сегменты

| Сегмент | Условие | Сценарий | Paywall |
|---------|---------|----------|---------|
| `skip_pending` | Есть незавершённая оплата | Пропуск | — |
| `skip_no_divinations` | Ни одного гадания | Пропуск (активация — отдельный поток) | — |
| `active_subscriber` | Безлимит / платные расклады / подписка | `send_gentle_nudge` | Нет |
| `expired_sub` | Был платный доступ, сейчас истёк | `send_expired_sub_reminder` | Да |
| `paywall` | Исчерпал бесплатные расклады, не платил | `send_no_divinations_reminder` | Да |
| `free_return` | Есть бесплатные расклады, уже гадал | `send_free_return_nudge` | Нет |

| Аудитория | Подписанные на карту дня, не заблокированные |

## 4. Напоминания об оплате

| Параметр | Значение |
|----------|----------|
| Расписание | Каждые 2 мин (если `PAYMENT_REMINDERS_ENABLED=true`) |
| Этапы | 10 мин → 1 ч → 3 ч после создания платежа |
| Пересечение с Пн/Чт | Пользователи с открытой оплатой **пропускаются** |

## Ручные рассылки

CLI: `python send_message.py --help`

| Флаг | Описание |
|------|----------|
| `--no-divinations` | Paywall (один шаблон, без сегментации) |
| `--gentle-nudge` | Мягкое напоминание платникам |
| `--free-return` | Напоминание с оставшимися бесплатными раскладами |
| `--expired-sub` | Истёкший доступ + paywall |
| `--payment-reminder` | Напоминание об оплате |
| `--broadcast` | Всем незаблокированным из БД |

## Техническая схема

```
APScheduler (Europe/Moscow)
│
├── 09:25 daily           → daily_card
├── :00/:30 10–20 daily   → activation_broadcast
├── :00/:30 10–20 Mon/Thu → divination_reminder_broadcast
└── every 2 min           → payment_reminders (optional)
```

### Поля БД (`max_users`)

| Поле | Назначение |
|------|------------|
| `activation_sent_at` | Когда отправлена welcome-активация |
| `last_div_reminder_broadcast_at` | Когда отправлена последняя Пн/Чт рассылка |
| `last_active_at` | Для расчёта персонального слота |

Миграция: `migrations/20260531_broadcasts.sql`
